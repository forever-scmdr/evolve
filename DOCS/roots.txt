Новый способ определения корневых айтемов при использовании CMS-интерфейса

При создании модели данных по умолчанию создается единственный айтем (не тип айтема) - корень,
владельцем которого считается группа common. Этот айтем не имеет пользователя-владельца (USER_ID = 0).
Также этот айтем не хранится в БД и имеет ID = 0. То, что корень не хранится в БД, позволяет уменьшить
количество записей в таблице ItemParent.

Для начала работы с интерфейсом CMS пользователь авторизуется. Определяюся его группа и ID.
Корневые айтемы для пользователя изначально нигде не определены.

Корневые айтемы для различных групп определяются в модели страниц и могут быть изменены в любой момент.
