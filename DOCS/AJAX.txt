Новые функции для AJAX-частей страниц.

1)	элементы (пока только div), которые должны получать содержимое из AJAX запросов, могут иметь атрибут
	ajax-href. Значение этого атрибута - ссылка на AJAX-страницу. Если надо показывать лоадер во время загрузки
	аякса - указывается атрибут ajax-show-loader=yes

2)  Формы, которые должны отправляться через AJAX, также можно помечать атрибутом ajax=true. Этот атрибут нужно
	применять к элементу form. Запрос будет отправляться на урл, указанный в атрибуте action формы. ID элемента,
	который закрывается лоадером во время отправки формы, надо указывать в атрибуте ajax-loader-id.

3)  Ссылки, которые должны отправляться через AJAX, также можно помечать атрибутом ajax=true. Этот атрибут нужно
	применять к элементу a. Запрос будет отправляться на урл, указанный в атрибуте href ссылки. ID элемента,
	который закрывается лоадером во время отправки формы, надо указывать в атрибуте ajax-loader-id.

4)  Инициализация этой функциональности происходит при загрузке страницы, а также при вставке нового содержимого из
	AJAX (только для элемента, в который происходит вставка). Поэтому для вставляемых частей функциональность также
	работает, можно вызывать вставку AJAX по цепочке (из одного AJAX другой AJAX).

5)  Для подключения функций AJAX надо подключить скрипт /admin/ajax/ajax.js Также в этом файле можно поменять картинку
	для лоадера.




Часть комментария из ajax.js. Принцип работы библиотеки

Обновление различных (нескольких) частей страницы с помощью одного AJAX запроса.

AJAX запрос отправляется на сервер, сервер возвращает часть HTML кода определенного формата.
Этот код содержит куски, которые нужно вставить в различные части исходной страницы.
Каждый кусок обернут в элемент (например, div), который соответствует определенному месту в исходной странице.
ID этого элемента соответствует ID элемента исходной страницы, содержимое которого надо заменить на полученное из AJAX запроса.
Признаком этого div'а-обертки является то, что он имеет атрибут class="result".

Пример:

Результат, возвращаемый запросом:

<div id="first" style="result">
		....... HTML 1 ......
</div>
<div id="second" style="result">
		....... HTML 2 ......
</div>
<div id="third" style="result">
		....... HTML 3 ......
</div>
<script id="JSON">
		....... JSON код ......
</script>

Исходная обновляемая страница:

....... HTML ......
<span id="first">
		....... HTML 1 ......
</span>
....... HTML ......
<div id="second">
		....... HTML 2 ......
</div>
....... HTML ......
<table id="third">
		....... HTML 3 ......
</table>
....... HTML ......

Метод берет из результата все элементы, style которых равен "result". Содержимое каждого такого элемента вставляется в
элемент исходной страницы с таким же ID (как в примере).

На время работы метода часть блоков исходной страницы может быть блокирована, ID блокируемых элементов надо указывать.
