Классы
1. Реестр связей (ассоциаций).
2. Реестр наследования айтемов.
	Вынести из матрицы смежности пользовательские айтемы, т.к. у них одиночное наследование. Поиск потомков
	осуществлять в 2 шага - сначала в матрице, потом в случае если наследник расширяем пользовательскими айтемами,
	то в списке пользовательских (ID родителя -> список пользовательских). Соответственно надо иметь признак
	"можно расширять пользовательскими" в айтеме и множество таких айтемов.
3. Тип айтема (переделать старый).
	* хранить типы параметров
	* хранить computed параметры со ссылками на базовые айтемы и их параметры
4. Реестр типов айтемов (есть старый).
	Удалить данные о сабайтемах
	Выступает как фасад для реестров связей и наследования.
	* получить тип айтема по названию и по ID
	* получить наследников айтема (extends)
	* получить предков айтема (extends)
	* получить связи айтема вверх (вложен в, одни к одному)
	* получить связи айтема вниз (в айтем вложено, один ко многим)
	* получить все связанные типы айтемов вместе во связями вверх
	* получить все связанные типы айтемов вместе во связями вниз
4. Реестр страниц (есть старый).
5. Механизм разбора файла модели данных на базе JSOUP.
	Переделать старый из SAX
6. Механизм разбора файла модели страниц на базе JSOUP.
	Переделать старый из SAX
7. Сделать служебный загрузчик айтемов (не ItemQuery)
	* загрузить ID айтемов-подписчиков
	* обечпечить необходимые возможности загрузки для потока обновления computed параметров
8. Переделать ItemQuery
