Классы
1. Реестр связей (ассоциаций).
2. Реестр наследования айтемов.
	Вынести из матрицы смежности пользовательские айтемы, т.к. у них одиночное наследование. Поиск потомков
	осуществлять в 2 шага - сначала в матрице, потом в случае если наследник расширяем пользовательскими айтемами,
	то в списке пользовательских (ID родителя -> список пользовательских). Соответственно надо иметь признак
	"можно расширять пользовательскими" в айтеме и множество таких айтемов.
3. Тип айтема (переделать старый).
	* хранить типы параметров
	* хранить computed параметры со ссылками на базовые айтемы и их параметры
4. Реестр типов айтемов (есть старый).
	Удалить данные о сабайтемах
	Выступает как фасад для реестров связей и наследования.
	* получить тип айтема по названию и по ID
	* получить наследников айтема (extends)
	* получить предков айтема (extends)
	* получить связи айтема вверх (вложен в, одни к одному)
	* получить связи айтема вниз (в айтем вложено, один ко многим)
	* получить все связанные типы айтемов вместе во связями вверх
	* получить все связанные типы айтемов вместе во связями вниз
4. Реестр страниц (есть старый).
5. Механизм разбора файла модели данных на базе JSOUP.
	Переделать старый из SAX
6. Механизм разбора файла модели страниц на базе JSOUP.
	Переделать старый из SAX
7. Сделать служебный загрузчик айтемов (не ItemQuery)
	* загрузить ID айтемов-подписчиков
	* обечпечить необходимые возможности загрузки для потока обновления computed параметров
8. Переделать ItemQuery


Интерфейс админа

Для каждого айтема добавить (можно в списке сабайтемов, можно на странице айтема, можно что-то там, что-то там, не знаю
как лучше):
1)  Скрыть/показать айтем. (чтобы скрытые айтемы как-то выделялись в списке айтемов слева)
2)  Изменить группу владельцев айтема (выбрать группу из списка и нажать ОК)
3)  Присвоить айтем пользователю (выбрать пользователя из списка возможно с поиском, нажать ОК)
4)  Скрыть/раскрыть файлы айтема (для общего анонимного доступа)

Создание связей
1)  Группировать доступные для создания сабайтемы по группам (каждая группа - своя ассоциация). Это касается верхней
	части списка айтемов, где создание (зеленые плюсы)
2)  Нажатие на плюс не в ассоциации "Первичная иерархия" вызывает окошко с выбором прикрепляемого айтема, аналогично
	существующим связям уровня или ассоциациям, только добавить еще возможность поиска.
3)  Группировать уже существующие сабайтемы по ассоциациям аналогично пункту 1.
4)  В списке сабайтемов добавить страницы и возможность выводить сабайтемы в обратном порядке
5)  Связи верхнего уровня заменить на что-то похожее на пункт 2 только наоборот, выбирается ассоциация и ищется новый
	родительский айтем. Возможно сначала ищется новый родительский, а потом выбирается ассоциация. Или рядом с каждым
	потенциальным родительским в списке выводить все возможные ассоциации для него.

Корзина
1)  Где-то должна быть ссылка на корзину
2)  В корзине должен быть скисок айтемов и ссылка восстановить
3)  Должна быть ссылка очистить.
